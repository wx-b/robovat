robovat.io package¶

robovat.io.episode_generation module¶

Run and generate episodes.

Copyright (c) 2018 Stanford University. Licensed under the MIT License (see LICENSE for details) Written by Kuan Fang

generate_episode(env, policy, num_steps=None, debug=False)[source]¶

Run and generate an episode.

Parameters

  • env – The environment.

  • policy – The policy.

  • num_steps – Maximum number of steps in each episode. None for infinite.

  • debug – True for visualize the policy for debugging, False otherwise.

Returns

The episode data as a dictionary of

’hostname’: The hostname of the computer. ‘timestamp’: The system timestamp of the beginning of the episode. ‘policy_info’: The policy information. ‘transitions’: A list of transitions. Each transition is a

dictionary of state, action, reward, and info.

Return type

episode

generate_episodes(env, policy, num_steps=None, num_episodes=None, timeout=30, debug=False)[source]¶

Run and generate multiple episodes.

Parameters

  • env – The environment.

  • policy – The policy.

  • num_steps – Maximum number of steps in each episode. None for infinite.

  • num_episodes – Maximum number of episodes. None for infinite.

  • timeout – Seconds to timeout.

  • debug – True for visualize the policy for debugging, False otherwise.

Yields

episode_index – The index of the episode. episode: The episode data.

robovat.io.hdf5_utils module¶

File IO utilities using HDF5.

The data element saved in HDF5 should be a dictionary. The types of each value should be in HDF5_DATA_TYPES.
